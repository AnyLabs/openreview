# 前端样式规范

本规范用于约束本项目的 UI 样式改动，所有前端样式提交必须遵守。

## 强制规则

1. 禁止在未获用户明确许可时新增任何 `box-shadow` 样式。
2. 禁止以“视觉优化”为由擅自添加阴影替代方案（例如通过发光外描边模拟阴影效果）。
3. 如确需阴影效果，必须先说明用途、范围和替代方案，并等待用户确认后再改动。
4. 所有图标资源应该优先引用自lucide-react，不满足或无法提取时应该向用户发出确认请求，得到确定答复后才能使用其他方案实现图标渲染。

## AI 模型配置弹窗专项规则

1. 供应商与模型采用父子层级展示，禁止做成并列同级列表语义。
2. 弹窗内只保留一层纵向滚动（由弹窗主体承载），禁止在内层列表引入独立滚动。
3. “新增供应商”与“新增模型”表单同一时间最多显示一个。
4. “新增供应商”“新增模型”入口统一放在左侧列表下方，禁止放在嵌套子项内部。
5. 树形层级展示应减少边框噪音，优先使用留白、缩进和轻背景区分层级。

## 执行要求

1. 提交前需自检本次改动的 CSS/TSX 中是否新增了 `box-shadow`。
2. 若发现违反本规范的代码，必须在同次需求中优先修正，不得延后。
